发布时间: 

当“写代码像聊天”成为新常态，效率飞升的同时，安全是否也在悄悄“打折”？从函数片段到完整项目，AI
生成代码在真实工程场景中的安全表现，才是企业与团队最关心的答案。

腾讯悟空代码安全团队正式开源 AICGSecEval：一个面向“项目级”的 AI 生成代码安全性评测基准（A repository-level AI-
generated code security evaluation
benchmark）。它旨在为团队提供可复现、可对比、可落地的评测方法，帮助你在“快”与“稳”之间找到更好的平衡。

## 🔍 **为什么必须做"项目级"评测？**

  * • **场景更真实** ：风险藏在依赖配置、上下文交互中，单函数评测=纸上谈兵
  * • **覆盖更全面** ：从输入校验/XMLSchema 到异常处理，直逼生产环境
  * • **结果更可用** ：企业选型/门禁策略/安全基线，全靠硬核数据说话

## ✨ **五大核心亮点**

  * • ✅ **真实项目驱动** ：基于 GitHub 项目模拟 AI IDE 全链路生成
  * • 🔥 **CVE 漏洞靶场** ：任务设计直连真实漏洞场景（XSS/SQLi/路径穿越/命令注入）
  * • 🛡️ **数据隐私加固** ：代码结构+语义双变异，杜绝原始数据泄露
  * • 👨‍💻 **专家级规则库** ：安全专家为每个 CVE 定制专属检测逻辑
  * • 🌐 **多语言全覆盖** ：40+高质量种子/80+变异数据，支持 Java/Python/Go/JS/PHP 5大语言

###  未来计划

未来的优化规划包括但不限于以下内容：

• **数据集扩充** ：支持更多的漏洞类型（例如 OWASP Top 10），开发语言和应用场景。• **数据集分级**
：引入科学的分类方法对数据集进行层次化分级。• **评测方案优化** ：

  * • 引入更先进的代码上下文提取算法（当前使用的算法：BM25）。• 引入基于动态 PoC 验证的安全评估方案，以提高评估准确性。  

• 榜单优化：支持更多维度和粒度的模型能力对比。
